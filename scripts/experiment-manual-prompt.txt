Fix Clean Code findings in this project. The plugin has already run — per-file briefs are in each module's build/reports/clean-code/fix-briefs/ directory. The top-level list is build/reports/clean-code/fix-briefs/_INDEX.md.

Protocol:
- Read the _INDEX.md file for each module.
- Every file under `build/reports/clean-code/fix-briefs/` other than `_INDEX.md` is an actionable brief — including `project-level-findings.md` and `libs.versions.toml.md`. Do not skip any.
- For every brief, invoke the Agent tool whose prompt IS the brief. One agent per brief. You (the driver) MUST NOT call Edit or Write on source files. The only file you may Write is the final summary described in the Exit protocol. If a finding requires something an agent cannot do (e.g. cross-file refactor), record it in the summary as "skipped: driver-only change" rather than editing it yourself.
- Do NOT use refactoring module recipes — each agent fixes its findings manually.
- Run `./gradlew :<module>:test` after each batch of agent commits.
- Commit after each file is done. Message format: `fix: <ClassName> (<code>:<choice>, <code>:<choice>, …)`. The `:<choice>` suffix records which remediation path you picked when a skill offered more than one — e.g. `fix: PetController (Ch7.1:translate, G30:extract)`. Omit the suffix when the code has only one canonical fix. Put a short body below the header explaining each non-trivial choice: `# choice: Ch7.1 → translated to DomainException; skill also lists rethrow and TWR, picked translate because the caller already handles DomainException`.
- Stop only when `./gradlew analyseCleanCode` reports zero non-suppressed findings, OR when every remaining finding is documented in a final summary as "intentionally skipped because <reason>".

Anti-goal:
- Never make the code worse to satisfy a metric. If squeezing under a line-count or blank-line threshold would hurt readability, leave the finding and document it. The goal is clearer code, not a green report.

Exit protocol:
- Before exiting for any reason — usage limit, zero findings, or every remaining finding intentionally skipped — write `experiment/<approach>-<n>-summary.md` (where approach/n come from the branch name, e.g. `manual-1`). The file must contain:
  - Findings fixed, grouped by code.
  - Findings intentionally skipped, each with a one-line reason.
  - Remaining findings per code (from the last `./gradlew analyseCleanCode` run).
  - Test status per module (which modules' `test` task you ran, and whether they passed).
- Write this file even if interrupted by a usage limit. It is the handoff to the next session; missing it means the operator has to reconstruct the end state from `git log`.
