# Skill: Project Conventions

## When to use this skill

- When working on any code in this project
- When the plugin has not identified a specific heuristic violation but
  you need guidance on the team's preferred patterns

This skill is a team-maintained document. The Clean Code plugin scaffolds
it on first run; the team fills it with project-specific conventions.

---

## How to use this file

Add your project's conventions below. These override or supplement the
patterns in the other skill files. For example:

- Approved libraries and frameworks (and which ones to avoid)
- Naming conventions specific to your domain
- Architectural boundaries (which modules can depend on which)
- Logging conventions (what to log at each level)
- Exception hierarchy for your domain
- Approved patterns for common operations (e.g. database access, HTTP calls)
